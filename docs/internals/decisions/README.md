# Architecture Decision Records

Use this folder for accepted decisions that future agents and maintainers should
not relitigate.

Each ADR should be short and focused on one decision. Use this shape:

```markdown
# 0001 Decision Title

Status: accepted
Date: YYYY-MM-DD

## Context

Why this decision was needed and what options mattered.

## Decision

The choice.

## Consequences

What this enables, what it costs, and what follow-up it implies.
```

Use statuses such as `proposed`, `accepted`, `superseded`, or `rejected`.
